# 실험 C - Q3 답변

## Adam이 초기 학습에 빠르게 수렴하는 이유

Adam은 학습 초기부터 안정적인 업데이트 크기를 보장하는 Bias Correction 기법을 사용한다.

**문제 상황:**
1차/2차 Moment를 계산할 때 초기값을 0으로 시작하면,
학습 초반 스텝에서 m과 v가 실제 값보다 0에 가깝게 편향(bias)된다.
이를 그대로 사용하면 초기 업데이트가 지나치게 작아진다.

**Bias Correction으로 해결:**

  m_hat = m / (1 - beta1^t)
  v_hat = v / (1 - beta2^t)

- t=1일 때 beta1=0.9이면: 1 - 0.9^1 = 0.1 → m을 10배로 보정
- t=10일 때: 1 - 0.9^10 = 0.65 → 보정값이 점점 1에 수렴
- 즉, 초기 스텝일수록 강하게 보정하고, 학습이 진행될수록 보정량이 자연스럽게 줄어든다

이 덕분에 Adam은 첫 번째 스텝부터 적절한 크기의 업데이트를 수행할 수 있다.

**파라미터별 적응형 학습률:**

최종 업데이트 크기는 다음과 같다:

  실제 업데이트량 = 학습률 × m_hat / (sqrt(v_hat) + epsilon)

- Gradient가 자주 크게 변하는 파라미터 → v_hat이 커짐 → 업데이트량이 작아짐 (안정성 확보)
- Gradient가 항상 작은 파라미터 → v_hat이 작음 → 업데이트량이 상대적으로 커짐 (학습 기회 보장)

결론적으로 Adam은 SGD처럼 모든 파라미터에 동일한 학습률을 적용하지 않고,
각 파라미터의 과거 Gradient 히스토리에 따라 개별적으로 학습률을 조정하기 때문에
초기부터 빠르고 안정적으로 최적점을 향해 수렴한다.
